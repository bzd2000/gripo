Data Entity Model

All entities use soft deletes (deleted_at). Subjects are the primary organizing unit.

subjects

id TEXT PK name TEXT NOT NULL pinned INTEGER (0) archived INTEGER (0) color TEXT created_at TEXT deleted_at TEXT
▼ one-to-many (subject_id FK) ▼

tasks

id TEXT PK subject_id TEXT FK text TEXT NOT NULL priority TEXT (must|should|if-time) status TEXT (todo|in-progress|done|blocked) category TEXT (delivery|admin|people|strategy|meeting|other) day TEXT (mon|tue|wed|thu|fri|anytime) today INTEGER (0) created_at TEXT completed_at TEXT deleted_at TEXT

open_points

id TEXT PK subject_id TEXT FK text TEXT NOT NULL context TEXT status TEXT (open|resolved|parked) resolved_note TEXT raised_at TEXT resolved_at TEXT deleted_at TEXT

waiting

id TEXT PK subject_id TEXT FK text TEXT NOT NULL owner TEXT NOT NULL asked_on TEXT due_by TEXT status TEXT (waiting|received|overdue|cancelled) notes TEXT deleted_at TEXT

notes

id TEXT PK subject_id TEXT FK content TEXT NOT NULL (markdown format) created_at TEXT updated_at TEXT deleted_at TEXT

meta

key TEXT PK value TEXT NOT NULL

Stores app state (e.g. week_of for rollover)
